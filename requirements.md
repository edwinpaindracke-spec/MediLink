Requirement

Functional Requirement  
•	User registration and login.
•	Stores doctor’s profiles and schedules. 
•	Appointment booking and management. 
•	SMS notification. 
•	Admin dashboard.

Non-Functional Requirement
• The system must sends SMS within 24Hours.
• Admin dashboard can be accessed only by admin.
• User data must be protected from unauthorized access.
• The system should support multiple users. 
• Only the user must see he/her bookings.
• The system should be user-friendly and accessible on mobile devices.
• The system should allow users to schedule his/her booking.
